# Project Milestone Checklist (M1 / M2 / M3)

This page tracks submission requirements for all project milestones in
one place.

------------------------------------------------------------------------

#  Milestone 01 -- Planning & Design

##  Proposal & Planning

-   [ ] Problem statement clearly defined
-   [ ] Target users identified
-   [ ] Scope defined (in-scope / out-of-scope)
-   [ ] Feature list categorized (Must / Should / Could)

##  Requirements

-   [ ] User stories written
-   [ ] Acceptance criteria included
-   [ ] Clear system workflow described

##  Design Artifacts

-   [ ] Wireframes completed
-   [ ] Data model planning drafted
-   [ ] Architecture overview included

##  Project Setup

-   [ ] GitHub repo created
-   [ ] Project board initialized
-   [ ] Wiki initialized

------------------------------------------------------------------------

#  Milestone 02 -- Front-End Implementation & Database Design

##  Working Front-End (in repo)

-   [ ] Core screens implemented (matching M1 wireframes)
-   [ ] Primary workflow functional in UI (mock data allowed)

##  Database Design Package (PDF)

-   [ ] ER diagram (or structured relational diagram)
-   [ ] SQL CREATE TABLE statements included
    -   [ ] Primary keys
    -   [ ] Foreign keys (where relevant)
    -   [ ] Constraints
    -   [ ] Sensible data types

##  Back-End Setup (in repo)

-   [ ] Project skeleton created
-   [ ] Runnable server entry point
-   [ ] Initial routes/controllers structure

##  Updated GitHub Projects Kanban

-   [ ] Tasks actively moved across columns
-   [ ] Clear ownership assigned
-   [ ] Meaningful task statuses

##  Updated Activity Blog / Wiki

-   [ ] At least two progress entries since M1
-   [ ] Decisions documented
-   [ ] Blockers & resolutions noted

##  README Updated

-   [ ] Steps to run front-end locally
-   [ ] Steps to run back-end locally
-   [ ] Brief team contribution summary

------------------------------------------------------------------------

#  Milestone 03 -- Full-Stack Integration, Testing & Final Demo

##  Fully Functional Full-Stack Application

-   [ ] End-to-end workflow works with database
-   [ ] Full CRUD implemented for core objects

##  Testing Summary Report (PDF, 2--5 pages recommended)

-   [ ] Test plan (features + test cases)
-   [ ] Results (pass/fail + notes)
-   [ ] Known issues / limitations (if any)
-   [ ] Automated test instructions (if used)

##  Deployment / Execution Instructions (README)

-   [ ] Clean machine setup steps included
-   [ ] Environment/config notes included

##  Final GitHub Projects Kanban Snapshot

-   [ ] Board reflects completed work
-   [ ] Remaining items identified (if any)

##  Final Activity Blog / Wiki Reflection

-   [ ] Summary of term progress
-   [ ] Major decisions explained
-   [ ] Contribution evidence included

##  Final Demo Video (max 7 minutes)

Must include: - \[ \] Project name + goal + target users - \[ \]
Architecture overview (front-end / back-end / DB) - \[ \] Feature
walkthrough (major workflow end-to-end) - \[ \] Brief testing
highlight - \[ \] Short closing (limitations + next steps)

##  Presentation Artifact

-   [ ] Slides (PDF) OR
-   [ ] One-page visual overview (architecture + features +
    responsibilities)
